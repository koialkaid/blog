这个博客的第一原则很简单：先把公开写作稳定地跑起来。

我不准备在第一版里加入登录、数据库、评论系统或者其他会增加维护负担的东西。博客应该先成为一个可靠的写作容器，而不是一个需要长期运维的小产品。

当前站点采用的路线是：

- Hugo 负责生成静态页面
- GitHub Pages 负责公开发布
- 内容以 Markdown 文件方式保存

这样的好处很直接：

1. 内容和配置都能跟着 Git 一起备份
2. 成本低，迁移成本也低
3. 写作和发布流程简单，不依赖后台系统

对个人博客来说，很多时候最难的不是开始，而是把维护成本压到足够低。只有足够轻，才更容易长期写下去。
