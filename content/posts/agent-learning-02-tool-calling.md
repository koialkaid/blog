## 只有语言还不够

如果一个 Agent 只能生成文字，它能做的事情其实很有限。

它可以解释，可以建议，也可以整理思路，但一旦任务需要读取文件、查询状态、运行命令、修改代码，它就必须接触外部世界。

这就是工具调用重要的地方。

## 工具让 Agent 接触真实环境

工具调用让 Agent 不只是“想”，而是可以做一些可验证的动作。

比如：

- 读取项目文件
- 执行构建命令
- 查看 Git 状态
- 请求网页内容
- 写入或修改文件

这些动作本身不复杂，但它们让 Agent 的回答不再只依赖猜测。

## 工具也会带来风险

工具越强，越需要边界。

一个能读文件的 Agent，要知道哪些文件可以读。一个能写代码的 Agent，要知道哪些目录可以改。一个能执行命令的 Agent，更应该知道什么时候不能继续尝试。

> 工具调用不是越多越好，而是越可控越好。

## 我现在的判断

学习 Agent 时，工具调用大概是绕不过去的一步。

但我更愿意先学会两件事：

1. 明确工具能做什么
2. 明确工具不应该做什么

只有这样，Agent 才更像可靠的执行者，而不是一个权限过大的聊天窗口。
