[深圳智能硬件公司招聘 AI Agent 工程师 （25k+） - 非我莫属 - LINUX DO](https://linux.do/t/topic/1966138)

工作地点： 深圳  
所属团队： 产品与技术团队

职位描述  
我们正在构建以 Agent 为核心的产品：它不仅响应用户输入，还能理解上下文、拆解任务，并在多个能力之间做决策与执行。  
这个岗位的核心，不是简单接入大模型，而是设计和实现一套可扩展、可演进的 Agent 系统——让 AI 能力在真实产品中稳定运行，而不是停留在 Demo。  
你将参与定义：一个 Agent 应该如何思考、如何调用工具、如何在复杂场景中持续工作。

主要职责  
设计并实现 Agent 系统架构（任务拆解、决策逻辑、Tool 调用、多步骤执行等）；  
构建大模型调用链路（Prompt 设计、上下文管理、多轮对话、记忆机制等）；  
设计 Tool Use 体系（API / function calling / 外部能力接入），提升 Agent 的可执行性；  
参与 AI 产品核心能力设计，将模型能力转化为稳定、可控的用户体验；  
优化 Agent 的稳定性与效果（响应质量、延迟、成本控制、失败兜底等）；  
与客户端、算法团队协作，推动 Agent 在真实业务中的落地与迭代；  
持续探索前沿 Agent 架构（multi-agent、planning、RAG 等）并评估落地价值。  
任职要求

1. AI / Agent 核心能力（必须）

有 LLM 应用开发经验（OpenAI / Claude / Gemini / Qwen 等）；  
理解 Agent 的核心机制（任务拆解、规划、Tool Use、多轮推理等）；  
熟悉 Prompt Engineering、Function Calling、上下文管理等关键能力；  
有实际 Agent / Copilot / 对话系统项目经验优先。
2. 系统与架构能力  
能独立设计复杂系统结构（而不仅是调用 API）；  
理解 Agent 在真实产品中的问题（延迟、稳定性、不可控性等）；  
后端开发经验（Python / Node.js 等），具备工程落地能力；  
熟悉服务架构（API 设计、状态管理、任务调度等）。
3. 产品理解与抽象能力  
能从用户问题出发，抽象为 Agent 可执行的流程；  
理解“能力 ≠ 体验”，能够将模型能力转化为可用产品；  
对复杂交互（对话、任务流、推荐等）有基本判断；  
能在不确定性中快速试验并收敛方案。
4. 工程与协作能力  
代码结构清晰，有良好的工程规范与可维护性意识；  
能与产品、设计、客户端工程师高效协作；  
具备强自驱力，能够独立推进复杂问题落地；  
持续关注 AI 技术发展，并能判断其实际价值。  
加分项  
有 multi-agent 系统设计经验；  
有 RAG（检索增强生成）、知识库或长上下文处理经验；  
使用过 LangChain、LangGraph、AutoGen、Coze 等框架；  
有 AI 产品从 0 到 1 落地经验；  
有性能优化或成本优化经验（如 token 控制、缓存策略等）。  
我们能提供  
核心位置： Agent 是产品中枢，你将参与其核心设计；  
问题深度： 面对的是“如何让 AI 真正可用”的系统级问题；  
成长空间： 从工程实现走向 AI 产品架构的路径；  
团队方式： 小团队快速迭代，直接影响产品方向。
