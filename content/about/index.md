你好，这里是 Koi's Blog。

这是一个从 **2026 年 4 月 9 日** 开始搭起来的个人博客。最开始只是看到 LINUX DO 上有人讨论「2026 年了，还有人用博客吗」，又看到一个极简双栏博客主题，觉得那种安静、克制、侧栏加内容区的形态很好看，于是就有了一个很简单的念头：

> 这个好看，用之。让我来部署试试。

后来它慢慢变成了现在这个样子。

## 这个博客基于什么

这个博客基于 **Hugo** 搭建。

我选择 Hugo，不是因为它最酷，也不是因为它功能最多，而是因为它足够直接：

- 内容是 Markdown 文件
- 页面由静态文件生成
- 不需要数据库
- 不需要长期维护服务器
- 迁移和备份都比较清楚

对一个个人博客来说，这种形式很安心。文章就放在仓库里，结构也能一眼看明白。

## 它是怎么部署的

这个站点托管在 **GitHub Pages** 上。

大致流程是：

```text
写内容
提交到 GitHub 仓库
GitHub Actions 自动构建 Hugo
部署到 GitHub Pages
```

所以前台看到的是一个纯静态博客。打开速度、备份方式和长期维护成本都比较稳定。

现在的线上地址是：

[https://koialkaid.github.io/blog/](https://koialkaid.github.io/blog/)

## 现在怎么写内容

一开始，这个博客是直接手改 Markdown 文件的。

后来为了降低写作门槛，又接入了 **Pages CMS**。它不是数据库后台，也不是 WordPress 那种完整内容系统，而是一个 Git-based CMS：在后台里填写标题、日期、摘要、标签和正文，保存后仍然回写到 GitHub 仓库里的 Markdown 文件。

也就是说，这个博客现在的内容链路是：

```text
Pages CMS
  ↓
Markdown 文件
  ↓
GitHub 仓库
  ↓
GitHub Actions
  ↓
Hugo 静态站点
  ↓
GitHub Pages
```

前台依然是静态博客，CMS 只是帮我少手写一些 front matter。

## 这里会放什么

目前内容主要分成几类：

- `文章`：相对完整一点的整理
- `日记`：短一点的记录和随手想法
- `系列`：围绕一个主题连续写的内容
- `待办`：博客相关的小计划和维护记录

我不太想把这里做成一个什么都放的地方。它更像是一个公开版本的写作空间：想清楚一点、愿意留下来的内容，再放到这里。

## 为什么还保留博客

博客在 2026 年当然不是最热闹的地方。

但它有一个优点：慢。

它没有即时反馈，也没有算法推送，更不需要每句话都立刻变成动态。对我来说，这反而适合留下那些需要慢慢整理的东西。

有些内容不一定重要，但写下来之后，至少比只留在脑子里更清楚一点。

## 这个站点想保持什么样

我希望它一直保持简单：

- 黑白为主
- 默认日间模式
- 夜间模式也尽量克制
- 左侧栏 + 主内容区
- 少一点花哨功能
- 多一点能长期写下去的稳定感

这里不会追求功能堆满。能帮助写作的东西会留下，影响写作的东西就尽量不加。

## 一点边界

这里默认是公开空间。

所以不会放太私密、太即时、牵涉别人边界，或者还没有整理清楚的内容。它更像是“整理之后愿意公开留下来的版本”，不是全部生活记录。

如果以后这个博客慢慢积累出一些能回看的文章和日记，那它就已经完成了自己的作用。
